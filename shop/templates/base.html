{% comment %} For category filtering: 1. The Button Click (base.html) You, the user, go to your website and click on the "Men" dropdown, then select "T-Shirts." The HTML for that link is href=" url 'product_category' 'men-t-shirts' ". Django turns this into a URL like /category/men-t-shirts/. The string men-t-shirts is a specific identifier, called a "slug." 2. The URL Match (shop/urls.py) Your web browser sends a request to the server for the URL /category/men-t-shirts/. Django's URL configuration has a pattern: path('category', views.product_category, name='product_category'). Django sees that your URL matches this pattern. It then takes the part of the URL that's in the slot (which is men-t-shirts) and saves it in a variable called category_slug. Django then calls the product_category function in your views.py file and passes it this category_slug variable. 3. The Database Query (shop/views.py) The product_category view function starts running. The code products = Product.objects.filter(category=category_slug) is executed. This is the most important step. It tells the database, "Go and find all the products where the category field in the database is equal to the value of category_slug." In this case, it's looking for all products where category is men-t-shirts. This is why it's essential that you write men-t-shirts in the admin panel. If you write something different, the database won't find a match, and no products will be shown on that page. 4. Displaying the Products (shop/product_list.htm) Finally, the view function takes the list of products it found (or an empty list if none were found) and the formatted category name ("Men T-shirts") and sends them to the product_list.html template. The template uses the category variable to display the title at the top. It then loops through the list of products and displays each one on the page. {% endcomment %}
